## 2. Top studies to run NEXT (ranked)

### #1 — OI-coverage census + 06-10 fetch-regime discontinuity audit  *(the only GREENLIGHT)*
- **Hypothesis:** The 06-10 DB rebuild created a sharp coverage discontinuity (OI-null 32.7%→6.1%, verified above) that silently changes the eligible population for every V/OI-gated classifier, so any study pooled over the full window mixes two regimes.
- **Gate:** N/A — infrastructure control, no edge claim, nothing to deflate. This is a **precondition feeder** that protects every downstream gate verdict from a population-mixing artifact.
- **Universe:** Hold constant — 117 roots present in BOTH regimes (controlled), 32 post-only roots held aside.
- **Effort:** ~hours (read-only EOD, ~5 queries). **Deliverable = a procedural rule, not a paper:** regime-tag every pooled row; restrict pooled studies to one coverage regime (or reweight / drop pre-06-10 OI-gated rows). **Encode the rule into the gate/trials-ledger directly.**
- **Why first:** cheap, done in an afternoon, and it is load-bearing for the trustworthiness of *every* other verdict (see §3).

### #2 — S1: Pan-Poteshman opening-vs-closing ΔOI discriminant  *(NEEDS_DATA, but the highest-quality real signal in the queue)*
- **Hypothesis:** Per-contract, market-neutral (cross-sectionally demeaned) net-positioning sign (opening vs churn, from settled ΔOI self-join) predicts forward 3d return. **Empirically real and monotonic Q1→Q5** (churn −0.030%/t=−1.81 → opening +0.072%/t=+3.96), and — critically — it **survives OUT of the AI/semis regime** (non-semis t=+2.85), the opposite of whale.
- **Gate:** PARTIAL. Survivable as a **market-neutral panel research finding on the STOCK leg** (~5,662 root×3d clusters → CPCV/MinTRL room). **REJECTS as a tradeable options strategy** — the +7bps stock edge → +0.64% option P&L vs 5.5% median option round-trip spread = −4.9%/trade. Two hard pre-registration locks: (a) must be the **demeaned per-contract panel**, NOT the naive pooled win-rate (which is 80% market beta, t=2.55→0.55 on demean — a textbook PBO trap); (b) economic null wired to the **underlying** leg.
- **Data gap:** needs **EOD stock prices for all 149 roots** (on-demand ThetaData pull, not cached) to express/score the equity leg. This is the one thing blocking it — and it's a single pull, not a new dataset.
- **Universe:** Current 149 is correct; breadth *helps* here (unlike whale). Filter 0DTE/weekly contamination.
- **Effort:** Medium. Pull stock EOD (~1 ThetaData session) → demeaned panel regression → gate on underlying leg. **~1–2 days.**
- **Honest expectation:** a clean, regime-robust **positive-on-equity / negative-on-options** finding. Publishable internally; do NOT expect a shippable options strategy.

### #3 — Pre-earnings IV term-structure inversion (vol-catalyst, side-label-INDEPENDENT)  *(NEEDS_DATA)*
- **Hypothesis:** Pre-earnings front-expiration ATM-IV hump (vs ~60d) is a label-independent vol-catalyst signal. Empirically visible (NVDA +0.22→+0.35 vol-pt hump on 02-24/25, collapses post-print).
- **Gate:** Likely a **clean NULL → SHADOW, not SHIP** (literature: hump is roughly fairly priced; long straddle is slippage-heavy). Its structural virtue: **immune to the "SIDE labels unreliable" finding** — pure priced vol, zero side dependence.
- **Data gap (the blocker):** **no historical earnings calendar exists anywhere in the repo** — `earnings_calendar.py` is forward-only Finnhub. Must source ~134 names' earnings dates **with BMO/AMC timing** externally and freeze before any gate run (timing determines which expiration brackets the event; naive nearest-expiry returns garbage). **Selection must BRACKET the earnings date, not pick min-DTE.**
- **Universe:** SUBSET — ~134 single names, exclude the ~15 ETF/index roots. More *calendar history* (quarters), not more roots, is what would help.
- **Effort:** Medium-high, gated entirely on sourcing + freezing the earnings calendar. **~2 days after calendar is in hand.**
- **Value:** corroboration of the De-Silva "flow-into-catalysts is negative-EV" thesis in a label-independent way. Worth it because the earnings calendar it unblocks is **reusable infra** for S6/S3-earnings and the shipped Batch-3a demote.

### #4 — Sector/regime-neutral WHALE test (THE pre-registered open question)  *(NEEDS_DATA)*
- **Honest assessment vs the alternatives: NO, it is no longer #1.** It is the highest-*intent* study but the lowest-*tractability* of the runnable group. Run it as a **falsification/closer, not a ship candidate.**
- **Hypothesis:** Does whale flow predict the within-same-sector basket-demeaned residual, and OUT of the AI-capex regime?
- **Two hard problems (both verified):** (1) In `chains.db` "whale" is an **EOD daily-aggregate liquidity tag** firing on 40–92% of liquid-name days (USO 92%, UNH 87%, GS 63%) — NOT the live sub-30s $3M ASK sweep; the defining ASK gate is deferred. (2) Held-out non-capex partitions collapse to 1–2 names (FIN=GS+BAC, ENERGY=USO+XLE+XOM), so a null is confounded with underpowering, not a clean efficacy rejection.
- **Gate:** Will produce **SHADOW/REJECT** — which technically answers the falsification (whale doesn't survive out-of-regime) but as a null-from-underpowering.
- **Effort:** Medium (spot is clean, OPRA-tape side confirmation ~5,500 fetches sub-hour). **~1–2 days.**
- **Recommendation:** Run it **once, explicitly framed as a closing falsification** with the data gaps named in the writeup — "whale-firing IS liquidity here (ρ=0.988) and there is no out-of-regime period, so whale alpha cannot be disentangled from sector-beta/liquidity in this universe." This closes the whale thread more cleanly than another noisy long/short backtest. **Do not present any passing result as an edge.**

**Why this ordering:** #1 protects all downstream verdicts (do it first, it's an afternoon). #2 is the only study with a *real, regime-robust, empirically-confirmed* effect (even if non-tradeable). #3 unblocks reusable earnings infra. #4 is the charter's flagship but is decision-grade-to-**close**, not to ship — sequence it last so it benefits from #1's regime tagging and #3's earnings calendar.

## 3. Data-quality audits to do FIRST (broken → invalidates downstream)

1. **06-10 coverage-regime tagging (= Study #1 above).** This is simultaneously the top study AND the top audit. **Until every pooled row is regime-tagged, the trials-ledger cumulative-N deflation is computed over an inconsistent population, so the gate's pooled DSR/MinTRL hurdles are subtly wrong.** Highest priority; do before any new gate run.

2. **Per-exp vs bulk `expiration=*` coverage consistency.** *(Flagged in the task as a candidate; not in the 31 proposals, so verify directly.)* The dataset doc says bulk-per-root-day fetch is "fast" and on-demand; confirm the cached `option_eod` rows for a root-day match a fresh bulk pull (no silently-dropped expirations/strikes). If the cache under-covers deep strikes relative to bulk, **every OI-geometry / king / GEX / skew study is computed on a truncated chain.** Quick spot-check: 3–5 root-days, cached strike-count vs bulk strike-count. **~1 hour.**

3. **OI settlement-timing assertion (oi = prior-day/morning-settled).** Already verified in multiple proposals (NVDA vol>500 with oi=0 is impossible under same-day settlement; vol=1202 on D → oi=1202 on D+1). **Any ΔOI study (#2, S4-converted, S5-buildup) depends on this** — keep the assertion as a regression check, don't re-derive.

4. **MNAR OI-null on high-priced roots.** OI-null is **missing-not-at-random** — concentrated in whole high-priced names (KLAC 86%, BKNG 77%, CDNS 74%, ASML 64%). Any OI-gated panel is survivorship-biased toward liquid contracts and **systematically blind in exactly the high-price/non-AI names** the sector-neutral test leans on. Name this in every OI study's limitations; it biases #4 specifically.

5. **0DTE / date==expiration contamination.** ~2.8–19.6% of traded rows are same-day-expiry; the highest-volume V/OI shocks are 0DTE where `oi(t+1)` cannot exist. Every self-join / forward-return study must drop these (most self-joins drop them mechanically, but assert it).
